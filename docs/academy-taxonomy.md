# Academy Taxonomy — Novice · Experienced · Expert

The academy is tiered by **skill level**, not price. A Novice buys Gold; an Expert browses Bronze — the tiers gate *content complexity*, not access.

## One-line definitions

| Tier | Who they are | Primary outcome |
|------|--------------|-----------------|
| **Novice** | Professional, never opened a terminal | Install Claude Code, do one real thing this week |
| **Experienced** | Comfortable with basics; wants real integration | Wire Claude Code into actual job, ship one automation |
| **Expert** | Treating Claude Code as core infrastructure | Architect team rollouts, custom MCP, hook-based governance |

## Course threads (cross-tier)

Each tier covers the same five threads but at different depth. A thread is a *vertical* through the academy: you can follow "MCP" from Novice to Expert in a straight line.

| Thread | Novice | Experienced | Expert |
|--------|--------|-------------|--------|
| **Mental model** | What Claude Code is and isn't | The 5 layers of a real workflow | Agentic systems architecture |
| **Daily practice** | First session, first edit, first CLAUDE.md | Skills, slash commands, hooks, multi-project | Team governance, shared skill libraries, hook-based safety |
| **Tool integration** | File system, one useful prompt | MCP: Gmail / Calendar / Notion / DB | Custom MCP servers, OAuth for teams, security model |
| **Decision-making** | Perspective Matrix (MERCURIO) + Six Dimensions (MARS) as one-page refs | `/dissent`, `/leverage`, `/feasibility`, `/timeline-layer` as slash commands | MoE orchestration, multi-domain research, parallel agent dispatch |
| **Scaling + safety** | Approve / deny fluency, the backup-first habit | Production patterns, team CLAUDE.md, hooks | Org-wide rollout, audit trails, compliance |

## Skill inventory per tier

### Novice — *Install and do one real thing*
7 modules, ~90 min total. Already written as source markdown in `content/start/`.
1. Mental model — what Claude Code is
2. Install (Mac / Win / Linux)
3. First session — ask it something useful
4. File permissions — edit a real file, revert
5. CLAUDE.md — fill-in template
6. One real use case (report / organize / email)
7. Next steps → Experienced

**MERCURIO / MARS port**: one page each — *Perspective Matrix* and *Six Dimensions* — framed as "thinking tools that make the AI more useful." No slash commands, no orchestration. Just the habit.

### Experienced — *Wire it into your job*
Equivalent to Silver cohort. Four weeks, one shipped automation.
1. Audit your real week — pick 3 automation targets
2. Skills + slash commands library (20+ real commands)
3. MCP wiring (Gmail, Calendar, Notion, databases) + security
4. Hooks, multi-project setup, team safety, capstone
5. MERCURIO ported — `/dissent`, `/perspective-matrix` commands
6. MARS ported — `/leverage`, `/feasibility`, `/six-dimensions`, `/timeline-layer`
7. Capstone shipping rubric

### Expert — *Run it as infrastructure*
Equivalent to Gold engagement or a new top-tier offering. Solo consultants, CTOs, ops leaders.
1. Agentic architecture — when to spawn parallel sessions
2. Custom MCP servers for internal tools
3. Hook-based governance — pre-commit, pre-tool-use, auditing
4. Team rollout playbook — shared skill libraries, training cadence
5. MoE orchestration — MERCURIO-style multi-expert synthesis
6. Domain-decomposition research — MARS-style parallel research + synthesis
7. Production-grade automation stacks (the 8-phase pipeline pattern from LUXOR/COURSEWARE)

## Assessment — where should a visitor start?

A three-question self-placement on the academy landing page:

1. "Have you used Claude Code before?" — No → Novice. Yes → continue.
2. "Have you set up an MCP connection to one of your tools?" — No → Experienced. Yes → continue.
3. "Have you built custom MCP servers, shared skill libraries, or rolled Claude Code out to a team?" — No → Experienced (stretch). Yes → Expert.

Under 30 seconds. No forms, no email gate on the placement itself.
